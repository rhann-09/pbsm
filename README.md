# Physics-Based SMPL-X Modelling (PBSM)

> [!WARNING]  
> This package is a WIP. All is subject to change without notice.

## Overview
PBSM is a custom Python package to enable experimentation with motion diffusion models. 
Currently work is being done on using Vroid Studio derived .vrm models to enact motions in a Genesis physics environement whilst using a Three.JS renderer in a browser.
Presently coupling between Genesis and Three.JS has been acheived, allowing character motions in Three.js to be driven by the physics in Genesis. 
The next steps are to map SMPLX joints and motions to the .vrm models and then implement a motion diffusion model to suply SMPLX motion inputs.

## Current Features & Implementation Status

### 1. VRM to Physics Pipeline (`VRM` Class)
* **glTF Parsing:** Extracts skeletal kinematic trees, global joint positions, mesh vertices, and Linear Blend Skinning (LBS) weights directly from `.vrm` files.
* **Automatic Mesh Segmentation:** Segments full-body meshes into distinct rigid parts based on dominant joint LBS influences.
* **Collision Geometry Generation:** Automatically generates precise 3D convex hulls (via `trimesh`) for each segmented body part to calculate mass and inertia.
* **MJCF Export:** Procedurally generates MuJoCo XML (`MJCF`) files representing the kinematic tree, collision geometries, and joint constraints for physics engines.

### 2. Physics Simulation Environment (`GenesisSim` & `Entity` Classes)
* **Engine Wrapper:** A streamlined management class for the `genesis` physics engine, handling initialisation, rendering options, and time-stepping.
* **Multi-Entity Support:** Robust `Entity` objects allow for the easy injection of multiple dynamic assets into the scene, including:
  * Articulated avatars (`MJCF`)
  * Custom 3D meshes (`STL`)
  * Standard primitives (`BOX`, `CYLINDER`, `SPHERE`)

### 3. Real-Time Web Visualization (`ThreeJS` Class)
* **Live Server Integration:** Automatically spins up thread-safe background HTTP and WebSocket servers to serve web assets and stream data without blocking the main Python physics loop.
* **Dynamic Scene Construction:** The frontend JavaScript dynamically builds the Three.js scene (primitives, materials, lights, and VRM loaders) on the fly based on a JSON configuration dict generated by the Python backend.
* **High-Fidelity Avatar Rendering:** Utilizes `@pixiv/three-vrm` to render materials, facial expressions (blendshapes), and physics-based secondary animations (SpringBones for hair/clothes).

### 4. Genesis-to-Three.js Coupling (`Bridge` Class)
* **Coordinate Synchronization:** Seamlessly translates the continuous `qpos` tensors from Genesis's Z-up coordinate system to Three.js's Y-up rendering space.
* **Automatic Grounding:** Calculates the resting offset vector between the VRM Render Root (feet) and Physics Root (pelvis) to prevent characters from visually hovering or clipping through the floor.
* **Dynamic Overrides:** Supports injecting high-level, non-physics data into the WebSocket payload mid-simulation, including:
  * Facial expressions and LookAt targets
  * Dynamic camera movements
  * Global scene variables (e.g., Wind force vectors that actively blow VRM spring bones)
